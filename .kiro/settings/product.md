# 恋语AI - 产品概要

## 一句话定位
恋语AI 是一个恋爱关系军师，帮用户看懂关系局势、判断推进时机、给出可执行的下一步动作。

## 核心价值
不是帮用户"回一句话"，而是帮用户回答：
- 我们现在到底到哪一步了？
- 对方是真的有兴趣，还是只是礼貌回复？
- 我现在该继续聊、该邀约、该放缓，还是该修复？

## MVP V1 主链路
1. 用户提供聊天上下文 → 2. 系统分析关系局势 → 3. 输出关系雷达 → 4. 判断是否适合推进 → 5. 给出下一步建议 + 可执行话术

## 核心页面（4个）
- **首页**: 引导进入关系分析，不是功能广场
- **关系分析输入页**: 收集聊天内容 + 背景信息 + 当前困惑
- **关系分析结果页**: 产品灵魂页面，展示关系雷达 + 推进建议 + 话术
- **聊天页**: 辅助页，承接追问和话术微调

## 核心 API
- `POST /api/ai/relationship-analysis` — MVP V1 唯一新增核心接口

## 技术栈
- 前端: 原生 HTML5 + CSS3 + JavaScript（无框架）
- 后端: Node.js + Express
- 数据库: MySQL / PostgreSQL
- AI: 多模型适配（当前默认 qwen-plus via 阿里云 DashScope）
- 跨平台: Capacitor (iOS/Android) + 微信小程序

## 产品红线
- 不做 PUA 工具，不鼓励操控/打压/欺骗
- 不做伪精确百分比预测
- 不做纯陪聊产品
- 先判断再建议，先稳情绪再做动作，给具体动作不给空话
